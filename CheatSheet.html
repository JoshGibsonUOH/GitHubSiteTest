GitHub Command Line Cheat Sheet

Note: anything inside of “[]” symbols should be altered to your use case without the symbols. E.G. “git branch [branch name]” would be “git branch FeatureABranch” in the command line.

Command Description
Initialising and Cloning Repositories
git init Initialise a new GIT repository in your current working directory
git clone [address to repo] Clone a remote repository to your device using its URL
Committing
git status Check the current status of any changes to the repository.
git add [file or directory] Sets the provided file or directory to be committed.
git add . Sets all files in the current directory to be committed.
git commit -m "Commit Message Here" Commits all added changes to the repository.
Branching
git branch lists all of the local branches.
git branch [branch name] Creates a new branch.
git switch [branch name] Swaps to a specific branch

Some versions of GIT may use git checkout [branch name] instead.
Merging
git merge [branch name] Merges the current branch into the repository
Remote Repositories
git remote add [name] [address to repo] Add a new remote repository
git push [remote name] [branch name] Push changes to a remote repository
git pull [remote name] [branch name] Pull changes from a remote repository
git fetch[remote name] Fetch changes from a remote repository without completing a merge
Stashing
git stash save "Message Here" Stash your changes
git stash list List all stashes
git stash apply Apply the latest stashed changes
git stash drop Drop the latest stash
git stash branch [branch name] Create a branch from a stash
Misc. Drop the latest stash
git log View the history of a repositories previous commits.
